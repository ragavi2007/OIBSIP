ℹ How this works

Password hashing: When you register, your password is passed through SHA-256 via the browser's native Web Crypto API and only the resulting hash is saved to localStorage. The original password is never written anywhere.

Session management: On successful login, a session object (username, email, timestamp) is stored in localStorage. Protected pages check for this object on load and redirect unauthenticated visitors straight back to the login page. Signing out clears the session immediately.
